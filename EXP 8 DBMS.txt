 THEORY (WITH SYNTAX)
A transaction is a logical unit of work in a database. All operations performed within a transaction are treated as a single unit. Changes made during a transaction become permanent only when they are committed. A transaction begins with an SQL statement and ends with either a COMMIT or ROLLBACK statement.
Transaction Control Language (TCL) commands are used to manage these transactions effectively.


🔸 1. COMMIT
The COMMIT command is used to permanently save all changes made during a transaction into the database. Once committed, the changes cannot be undone.
Syntax:
COMMIT;
Example:
COMMIT;


🔸 2. SAVEPOINT
A SAVEPOINT is used to divide a large transaction into smaller parts. It acts as a marker within a transaction so that we can rollback to a specific point instead of the entire transaction.
Syntax:
SAVEPOINT savepoint_name;
Example:
SAVEPOINT xyz;


🔸 3. ROLLBACK
The ROLLBACK command is used to undo changes made during a transaction. It can either undo the entire transaction or rollback to a specific savepoint.
Syntax:
ROLLBACK;
ROLLBACK TO savepoint_name;
Example:
ROLLBACK;
ROLLBACK TO SAVEPOINT xyz;


🔹 CONCLUSION
Thus, we have studied and implemented transaction control commands such as COMMIT, SAVEPOINT, and ROLLBACK. These commands help in managing database transactions efficiently by ensuring data consistency, allowing partial rollback, and making changes permanent when required.
